How the transport detection works

The JS opens an EventSource and starts a 3-second probe timer. The server sends a probe SSE event immediately on connection.
• Direct proxy (port 3000): bytes forwarded chunk-by-chunk → probe arrives in <50 ms → timer cancelled → HTTP-STREAMING confirmed.
• Buffering proxy (port 9090): response buffered until complete (never for SSE) → probe never arrives → 3-second timer fires → EventSource closed → long-polling loop starts → HTTP-POLLING confirmed.
When the buffering proxy is removed, the client reconnects and gets streaming automatically. Zero code changes.
